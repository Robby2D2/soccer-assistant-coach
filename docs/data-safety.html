Data Safety — Recommended answers

This page mirrors the guidance in store/DATA_SAFETY.md. Review and adapt if you add analytics or cloud features.

Summary

  • Collects personal data: Yes — user-provided team/player names and notes are stored locally.
  • Shared with third parties: No by default — data is only shared when the user explicitly exports or shares a file.
  • Analytics / Ads: No (unless you add analytics SDKs later).

Types of data

  • Personal info: Yes — player and team names, notes entered by the user (stored locally).
  • Photos / Media: Optional — users can attach photos via the image picker; stored locally.
  • Files: Yes — CSV export/import files are created by the user.

Purposes

  • App functionality: Yes — used to provide core app features (rosters, lineups, exports).
  • Analytics: No
  • Advertising: No

Notes for Play Console

In the Play Console, mark "Yes" for collects personal data and explain that the data is stored locally and only shared on user action (export/share). Mark "No" for ads and analytics if you have none.

Hosted from the repo. Update if you add server-side backups, analytics, or cloud sync.
