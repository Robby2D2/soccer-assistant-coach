# Releases & Store Publishing — learnings (April–May 2026)

Current release procedure lives in [`docs/RELEASING.md`](../../docs/RELEASING.md). This file holds
the hard-won debugging knowledge behind it.

## Fastlane / WSL (May 2026)

- **`fastlane bump`/`create_release` push fails silently from WSL on this machine.** WSL git calls
  `git-credential-manager.exe` under `/mnt/c/Program Files/...`; the space in the path breaks the
  shell (`/mnt/c/Program: not found`). Fastlane prints "Successfully committed" but the commit +
  tag stay local, so no release workflow fires. Always verify from PowerShell with
  `git ls-remote --tags origin v<version>`; if empty, `git push origin main && git push origin v<version>`.
  Permanent fix: `docs/wsl-git-credentials.md`.
- **The Android `:release` lane must build the AAB itself** (`lane :release { build; deploy }`).
  CI invokes `fastlane release` directly on tag push with no separate build step; when the lane
  only called `deploy`, every release failed with "Could not find aab file at path".
- The iOS lane is asymmetric on purpose: CI builds the IPA as a separate macOS workflow step, so
  the iOS `:release` lane only uploads.
- Fastlane is vendored (`vendor/bundle/ruby/3.2.0`) and only runs from WSL via `bundle exec`.
  WSL project path: `/mnt/c/Users/rdane/Documents/Projects/soccer-assistant-coach`.

## iOS CI (May 2026)

Full checklist in [`ios_setup.md`](ios_setup.md) (one-time steps complete as of 2026-05-18). Key
debugging facts:

- **`flutter build ipa` hangs silently** unless `match` installs the cert into a named keychain
  (`create_keychain` + `keychain_name`/`keychain_password`) **and**
  `security set-key-partition-list -S apple-tool:,apple: -s -k …` runs before the build. xcodebuild
  runs outside fastlane's session and otherwise waits forever on a keychain prompt.
- Use **`flutter build ipa --export-options-plist=ios/ExportOptions.plist`** as a standalone
  workflow step, not fastlane `build_app` (same hang + CocoaPods ruby conflicts inside `sh()`).
- Runner: **`macos-14`**. `macos-13` queues >45 min; `macos-latest`/15 had 6-hour silent hangs.
- App Store Connect API key must have the **Admin** role — Developer role can't create
  Distribution certs, and Match fails with a misleading auth error.

## App Store submission tooling (May 2026)

- `bundle exec fastlane ios metadata` uploads metadata + screenshots (no submit);
  `ios submit` submits the latest TestFlight build for review.
- Screenshot naming for deliver: filename prefixes `iphone69` / `iphone67` select the 6.9" / 6.7"
  device sizes. Generated by `store/generate_assets.py`.
- Submission answers: export compliance `false` (standard HTTPS only), IDFA `false`.
- Ignorable deliver warnings: "Error fetching app store review detail — No data" (first run per
  version) and "Skipping release_notes… this is the first version" (v1.0 only).

## Play Store compliance (April–May 2026)

- **`USE_EXACT_ALARM` is rejected** for non-alarm/calendar apps. Use `SCHEDULE_EXACT_ALARM`
  (user opt-in) with the existing inexact-alarm fallback in `notification_service.dart`.
- Store assets regenerate with `python -X utf8 store/generate_assets.py` (Pillow).

## Android CI toolchain (May 2026)

Durable version/config rules are in `.agents/ARCHITECTURE.md`'s decision log (Jetifier off,
Built-in Kotlin, Gradle 8.13 / AGP 8.11.1 / Kotlin 2.2.20, JVM 17). Failure signatures worth
remembering:

- **Gradle heap OOM kills the runner silently** — step shows `in_progress` forever with no logs.
  Cause: `Xmx8G + MaxMetaspaceSize=4G` on a 7 GB runner. Cap at `Xmx2g -XX:MaxMetaspaceSize=512m`.
- **Jetifier OOMs on Flutter's engine JARs** (already AndroidX) — keep
  `android.enableJetifier=false`.
- "Inconsistent JVM Target Compatibility" after removing `kotlinOptions {}`: Kotlin 2.2 defaults
  jvmTarget to the JDK (17); set `compileOptions` to `VERSION_17` to match.
- Gradle 8.14.0 does not exist at the distribution URL — let the AGP error name the real minimum.
